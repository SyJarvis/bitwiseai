## 设计原则

### 核心理念
- **零基础小白导向**：假设用户完全不了解该领域，一切从零开始
- **角色代入感**：用户是故事的主角，不是学生或考生
- **自然对话**：选项应像日常对话中的自然回应，而非考试题目
- **引导思考**：通过场景和对话引导用户提问、思考，而非直接给出问题
- **日常化场景**：设计生活中的对话内容，让用户感觉像在和朋友聊天
- **知识传递**：通过情节推进揭示知识点，避免说教式讲解
- **多分支归一**：提供选择自由度，但最终引导至相同的学习目标

### 避免的常见错误

❌ **一问一答式（做题模式）**：
```json
{
  "content": "Python 中用于定义变量的关键字是什么？",
  "options": [
    {"text": "A. var", "nextNodeId": "2"},
    {"text": "B. let", "nextNodeId": "3"},
    {"text": "C. 直接赋值", "nextNodeId": "4"}
  ]
}
```

❌ **直接测试知识点**：
```json
{
  "content": "现在我们来测试一下：如果要保存一个数字 42，应该使用什么类型？",
  "options": [
    {"text": "字符串", "nextNodeId": "2"},
    {"text": "整数", "nextNodeId": "3"}
  ]
}
```

❌ **假设用户已有知识**：
```json
{
  "content": "你知道的，变量就是用来存储数据的，所以我们定义它...",
  "options": [...]
}
```

✅ **角色代入式（自然对话）**：
```json
{
  "content": "导师手中拿着一个发光的水晶，温和地问道：\"看这个水晶，是不是很漂亮？你想给这个水晶起个什么名字呢？\"",
  "options": [
    {"text": "\"就叫'水晶'吧，简单直接\"", "nextNodeId": "2"},
    {"text": "\"我想叫它'星光'，因为它会发光\"", "nextNodeId": "3"},
    {"text": "\"其实我还没想好...\"", "nextNodeId": "4"}
  ]
}
```

✅ **引导用户提问**：
```json
{
  "content": "你看着手里这个会发光的水晶，突然想到：如果以后我想再找到它，应该怎么做呢？",
  "options": [
    {"text": "问导师\"有没有办法记住它的位置？\"", "nextNodeId": "2"},
    {"text": "自己在心里默默记下位置", "nextNodeId": "3"}
  ]
}
```

✅ **日常化对话场景**：
```json
{
  "content": "午餐时间，你和同学一起在食堂吃饭。同学好奇地问：\"那个发光的水晶，到底是什么啊？\"",
  "options": [
    {"text": "\"我也不太清楚，导师说是个很神奇的东西\"", "nextNodeId": "2"},
    {"text": "\"好像叫变量，能保存各种东西\"", "nextNodeId": "3"}
  ]
}
```

## 剧情结构设计

### 零基础小白导向的剧情设计

**关键原则**：
1. **一切从零开始**：不假设用户了解任何相关概念
2. **用生活类比**：用日常熟悉的事物解释抽象概念
3. **循序渐进**：每个知识点都建立在用户已经理解的基础上
4. **允许困惑**：用户可以表达"我不明白"，这是学习和探索的开始
5. **鼓励好奇**：设计场景让用户自然产生疑问，而非被动接受知识

**设计流程示例**：

| 阶段 | 目标 | 内容示例 |
|------|------|---------|
| **探索阶段** | 引导用户产生疑问 | "这个水晶为什么会发光？它有什么用？" |
| **类比阶段** | 用生活场景类比 | "就像给一个盒子贴标签，以后就能找到它" |
| **实践阶段** | 在场景中尝试使用 | "我们来试一试，把火元素放进这个盒子里" |
| **反思阶段** | 总结和自然引出新知识 | "刚才我们学会了保存名字，那如果要保存数字呢？" |

### 日常对话场景设计

在日常对话中自然融入知识点，让用户感觉像在和朋友聊天：

**场景 1：和同学闲聊**
```json
{
  "content": "课间休息，你在走廊遇到了同学小明。小明好奇地问：\"听说你今天学了什么魔法？我看你一直在摆弄那个水晶。\"\n\n你笑着把水晶拿给他看：\"其实我也刚学，这个水晶能记住我们放进去的东西。\"\n\n小明挠挠头：\"这是什么原理啊？是不是很复杂？\"",
  "options": [
    {"text": "\"我也说不清楚，感觉就像是给东西起了个名字\"", "nextNodeId": "2"},
    {"text": "\"导师说叫变量，不过我也还在学习中\"", "nextNodeId": "2"}
  ]
}
```

**场景 2：和导师交流**
```json
{
  "content": "晚自习后，你有些疑惑地找到导师：\"老师，今天学的东西我有点不明白...\"\n\n导师温和地看着你：\"没关系，有什么不明白的地方尽管说，我慢慢解释。\"\n\n你组织了一下语言：\"就是这个'变量'的概念，我总觉得有点抽象...\"",
  "options": [
    {"text": "\"能不能举个例子让我理解得更清楚一点？\"", "nextNodeId": "2"},
    {"text": "\"为什么要给它起名字，直接用不就行了吗？\"", "nextNodeId": "3"}
  ]
}
```

**场景 3：独自思考**
```json
{
  "content": "回到宿舍，你躺在床上，回想着今天学到的东西。窗外的月光洒在桌子上，你的水晶静静躺在那里，微微发着光。\n\n你坐起来，拿起水晶自言自语：\"原来如此，就像给朋友起名字一样...\"",
  "options": [
    {"text": "突然想到一个问题，拿起笔在纸上记下来", "nextNodeId": "2"},
    {"text": "感觉理解了，安心地睡去", "nextNodeId": "2"}
  ]
}
```

### 故事背景选择
根据学习主题选择合适的故事场景：

| 学习主题 | 推荐故事背景 | 零基础切入点 |
|---------|------------|-------------|
| 编程基础 | 魔法学院、未来科技城 | 从生活中最简单的"记住"开始 |
| 数学公式 | 星际航行、建筑设计 | 从"为什么需要计算"开始 |
| 历史事件 | 时空旅行、宫廷秘史 | 从"亲身经历"开始 |
| 科学原理 | 实验室探险、自然探索 | 从"观察现象"开始 |

### 节点类型

1. **引入节点**：建立故事背景，引入学习主题
   ```json
   {
     "id": "start",
     "content": "# 魔法学院入学考试\\n\\n你站在魔法学院宏伟的大门前，手中紧握着录取通知书...",
     "options": [...]
   }
   ```

2. **知识节点**：通过情节传递核心知识点
   ```json
   {
     "id": "lesson1",
     "content": "老师指着黑板上的魔法阵：\"观察这个变量魔法阵，我们先给它起个名字...\"",
     "options": [...]
   }
   ```

3. **分支节点**：提供多个选择，形成不同剧情路径
   ```json
   {
     "id": "choice1",
     "content": "现在你可以选择不同的魔法路径...",
     "options": [
       {"text": "选择元素魔法", "nextNodeId": "path1"},
       {"text": "选择时间魔法", "nextNodeId": "path2"}
     ]
   }
   ```

4. **结局节点**：所有分支汇聚，总结学习成果
   ```json
   {
     "id": "ending",
     "content": "恭喜你完成了魔法学院的课程！你现在已经掌握了变量、数据类型和控制流...",
     "isEnding": true
   }
   ```

## 知识点融入方法

### 技巧 1：类比法
将抽象概念映射为故事元素

**示例**：编程变量 → 魔法容器
```json
{
  "content": "导师拿出一个空的水晶瓶：\"这就是变量，就像这个瓶子，我们可以往里面装入不同的魔法元素...\"",
  "options": [
    {"text": "\"装入火元素\"", "nextNodeId": "fire"},
    {"text": "\"装入冰元素\"", "nextNodeId": "ice"}
  ]
}
```

### 技巧 2：情境化
将知识点嵌入具体情境

**示例**：物理公式 → 飞船轨道计算
```json
{
  "content": "飞船电脑显示当前速度为 3000 m/s，需要计算 60 秒后的位移。你看着屏幕上的公式：\\n\\n$$s = v_0t + \\frac{1}{2}at^2$$\\n\\n\"如果加速度是 10 m/s²...\"",
  "options": [
    {"text": "我来计算...", "nextNodeId": "calc1"},
    {"text": "让电脑来算", "nextNodeId": "calc2"}
  ]
}
```

### 技巧 3：渐进式揭示
通过多节点逐步展开知识点

**节点序列示例**：
1. 引入概念：变量是什么（魔法容器）
2. 基本操作：赋值和使用（填充和使用魔法）
3. 类型转换：类型变化（元素转化）
4. 实际应用：综合任务（魔法挑战）

## 选项设计规范

### 零基础小白导向的选项设计

**核心原则**：
1. **允许表达困惑**：选项中应包含"我不明白"、"不太清楚"等选择
2. **模拟真实反应**：选项应反映一个完全零基础的用户的真实想法
3. **引导好奇**：通过选项引导用户自然产生疑问和思考
4. **避免术语**：选项中不应直接使用专业术语或代码
5. **情感共鸣**：选项应包含情感表达，如好奇、困惑、兴奋等

### 日常对话式选项

选项应像日常对话中的自然回应，填补对话的空白：

**标准格式**：
```json
{
  "content": "导师温和地看着你：\"看这个水晶，是不是很漂亮？你想给这个水晶起个什么名字呢？\"",
  "options": [
    {"text": "\"就叫'水晶'吧，简单直接\"", "nextNodeId": "2"},
    {"text": "\"我想叫它'星光'，因为它会发光\"", "nextNodeId": "3"},
    {"text": "\"其实我还没想好...\"", "nextNodeId": "4"}
  ]
}
```

**渲染效果**：
用户选择第一个选项后，页面显示：
> 导师温和地看着你："看这个水晶，是不是很漂亮？你想给这个水晶起个什么名字呢？" "就叫'水晶'吧，简单直接"

### 引导思考的选项

通过选项引导用户自然产生疑问，而非直接给出问题：

```json
{
  "content": "你看着手中这个会发光的水晶，突然想到一些问题：",
  "options": [
    {"text": "\"如果以后我想再找到它，应该怎么做呢？\"", "nextNodeId": "2"},
    {"text": "\"这个水晶能保存多少东西呢？\"", "nextNodeId": "3"},
    {"text": "\"如果把火元素放进去，它会改变吗？\"", "nextNodeId": "4"}
  ]
}
```

### 表达困惑的选项

允许用户表达不理解，这是学习过程的一部分：

```json
{
  "content": "导师解释了半天，你还是有点似懂非懂...",
  "options": [
    {"text": "诚实地表达：\"老师，我还是不太明白...\"", "nextNodeId": "explanation"},
    {"text": "不好意思说，只能点头：\"哦...好的\"", "nextNodeId": "nod"},
    {"text": "鼓起勇气：\"能不能举个例子让我理解得更清楚一点？\"", "nextNodeId": "example"}
  ]
}
```

### 选项数量控制
- 每个节点 2-3 个选项为宜
- 选项过多会增加用户决策负担
- 确保每个选项都有明确的剧情意义

### 选项区分度（零基础版本）

避免测试知识点，而是反映不同的思维路径：

**❌ 错误示例（测试知识点）**：
```json
{
  "content": "现在需要将字符串 '123' 转换为数字...",
  "options": [
    {"text": "使用 int('123')", "nextNodeId": "correct"},
    {"text": "使用 str(123)", "nextNodeId": "wrong"},
    {"text": "直接用 123 就行了", "nextNodeId": "wrong"}
  ]
}
```

**✅ 正确示例（反映真实想法）**：
```json
{
  "content": "你看到 '123' 这个数字被引号包着，突然想到：",
  "options": [
    {"text": "\"为什么数字还要加引号？有什么区别吗？\"", "nextNodeId": "explain"},
    {"text": "\"引号包起来就是字符串，不加就是数字...应该是对的吧？\"", "nextNodeId": "guess"},
    {"text": "\"其实我有点混淆了，能不能再讲讲？\"", "nextNodeId": "ask"}
  ]
}
```

## DAG 编排技巧

### 基本结构

```
      [开始]
        |
     [节点1]
     /    \
  [节点2A] [节点2B]
     |      |
  [节点3A] [节点3B]
     \    /
     [结局]
```

### 分支策略

1. **知识平行分支**：同一知识点的不同视角
   ```
   分支A：从实践角度（编写代码）
   分支B：从理论角度（理解原理）
   ```

2. **难度分级分支**：适合不同水平的学习者
   ```
   分支A：提供详细提示（新手友好）
   分支B：挑战更高难度（进阶用户）
   ```

3. **角色视角分支**：不同角色的体验
   ```
   分支A：扮演学生（学习知识）
   分支B：扮演老师（设计教学）
   ```

### 归一设计

所有分支应在最终结局节点前汇聚，确保：
- ✅ 用户获得完整的学习体验
- ✅ 核心知识点都被覆盖
- ✅ 结局给出统一的总结和鼓励

**反例**（不推荐）：
```
分支A → 结局A（掌握变量）
分支B → 结局B（掌握循环）  // 分散了学习目标
```

**正例**：
```
分支A → [学习变量] ─┐
分支B → [学习循环] ─┼→ [综合应用] → [结局：掌握基础]
分支C → [学习函数] ─┘
```

## 完整示例

### 主题：Python 变量与数据类型（零基础版本）

**故事背景**：魔法学院的新生，完全不懂编程，从零开始学习"变量"的概念

**设计思路**：
- 不直接教授"变量"这个术语，而是用"记住名字"这个日常概念引入
- 允许用户表达困惑和真实想法
- 通过日常对话场景让用户自然带入角色

**剧情数据**：
```json

    },
```
